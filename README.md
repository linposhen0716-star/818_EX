# 818 進銷存管理系統 (818_EX)

這是一個專為「818 早餐店」設計的簡易進銷存管理系統。透過 Python 自動計算銷售量，並在 Web 介面上提供直觀的盤點與進貨建議。

## 系統功能
1.  **自動計算銷售**：讀取 POS 匯出的 Excel，自動換算材料耗用量。
2.  **盤點進貨介面**：美觀的網頁介面，分類顯示各廠商產品，並根據銷量與庫存給予進貨建議。
3.  **數值自動進位**：系統內的「銷量」與「建議進貨量」均自動套用 **0.5 為單位** 的無條件進位邏輯（例如 1.3 變 1.5），符合實際進貨習慣。
4.  **一鍵收納**：提供快速摺疊所有廠商清單的功能，方便快速檢視。
5.  **本地儲存**：提供「存檔」按鈕，將盤點進度即時儲存於瀏覽器，下次開啟會自動載入。
6.  **Excel 匯出**：獨立的「匯出Excel」按鈕，產生 `.xlsx` 進貨清單，方便廠商對帳與列印。

## 檔案說明
- `index.html`: 管理介面主網頁。
- `JavaScript/main.js`: 核心邏輯與廠商資料庫。
- `update_sales_strict.py`: **重要工具**。每週跑一次，用來更新 `main.js` 裡的銷售數據。
- `mapping_dict.json`: 關鍵設定檔。對應 POS 產品名稱與系統內的正式名稱。
- `import_data.py`: 當需要大批次更新廠商清單時使用。

## 使用步驟
1.  將 POS 匯出的 Excel 檔案放入 `818_餐點銷售` 的對應資料夾。
2.  執行 `update_sales_strict.py` 更新 `main.js` 中的銷售量。
3.  點擊 `index.html` 進行盤點作業。
4.  **填寫庫存**：根據實際狀況填入庫存，系統會動態顯示建議進貨量。
5.  **點擊「存檔」**：將填寫好的數據暫存在瀏覽器。
6.  **點擊「匯出Excel」**：下載最終的進貨對帳單。
